# Сквозная атрибуция Директа через бот + кодовые слова (идея Андрея, 26.06.2026)

## Контекст / проблема
Реклама в Директ даёт живую переписку (человек на виду), НО у директ-объявлений нет ссылки → UTM не зашить → эти люди не попадают в нашу CRM/дашборд (подтверждено: 0 из 6 рекламных директ-лидов нашлись в базе 216). Ручное отслеживание Директа возможно, но при потоке людей не масштабируется — устанем.

## Идея Андрея (суть)
Подключить к Директу бота (ChatPlace / BotHelp / иной), который:
1. Считывает **старт диалога** в Директе.
2. Ловит **кодовое слово**, которое человек пишет (слово зашито в креатив — призыв «напишите слово X»).
3. По слову понимает: (а) это пришло **по рекламе**, (б) это **лид**, (в) из **какого конкретно креатива** (в каждый креатив — своё уникальное слово).
4. Сообщает в CRM: начался диалог, заводит лида с источником = этот креатив, запускает отчёт.
5. Даёт связку: **потрачено на креатив ↔ сколько по слову пришло в Директ ↔ сколько из них дошло до оплаты**.
6. Бот видит сам диалог; диалог можно подтягивать в CRM для анализа.

## Оценка реализуемости (честно)
**В целом адекватно и это стандартный паттерн.** Что верно:
- Автоматизация Директа через инструмент с доступом к Instagram Messaging API (BotHelp — УЖЕ подключён для лид-магнита `@brochurefinderbot`; ManyChat; ChatPlace) — рабочая практика. Нужен Professional/Business аккаунт + привязка к FB-странице.
- Триггер по ключевому слову в Директе и роутинг по слову → тег источника/креатива — базовая функция этих инструментов.
- Разные слова на разные креативы → атрибуция по креативу — реализуемо.
- Вебхук из бота в CRM на событие (старт диалога / совпало слово / оплата) → завести/обновить лида с источником — реализуемо.
- Подтянуть текст диалога в CRM для анализа — реализуемо через API/вебхук.

**Уточнения / что докрутить:**
1. **Для ПЛАТНЫХ click-to-Direct объявлений (цель «Сообщения») Meta сама передаёт `ref`/ID объявления** в payload первого сообщения — инструмент (BotHelp/ManyChat) может поймать источник **автоматически, без кодового слова**. Это надёжнее слов. → Для таргета использовать авто-захват ad-referral; кодовые слова — основной метод для ОРГАНИКИ (рилсы в ленте, где ссылки нет — совпадает с Фазой 2 ТЗ).
2. **Кодовое слово зависит от того, напишет ли человек его точно.** Часть напишет «привет» или вопрос → атрибуция по слову ловит только подмножество (тех, кто следует CTA). Поэтому: платный таргет → авто-referral; органика → слово; недостача допустима, но честно её учитывать.
3. **Сторона расхода всё равно ручная.** Бот считает лидов-по-слову, но СКОЛЬКО потрачено на креатив берётся из настройки Насти. Без per-creative spend связку «деньги ↔ лиды ↔ оплаты» не замкнуть → это вопрос Насте.
4. **Инструмент:** логичнее брать УЖЕ подключённый **BotHelp** (он и так синкается в дашборд ежечасно), а не плодить новый. ChatPlace в проекте Андрея используется под его СОБСТВЕННУ�ю IG-воронку (@brezgin), для AI Workshop основной — BotHelp. Решение за Арташесом по факту доступов.
5. Политика Messaging API (окно 24ч на авто-сообщения) — для этого сценария ок.

## Итоговая воронка после реализации (Директ-ветка станет видимой)
креатив (своё слово / ad-referral) → старт диалога в Директе (бот ловит) → лид в CRM с источником=креатив → события диалога → выбор оплаты → оплата → CPL и CAC по каждому креативу.
