# Методология создания лендинга — полный план

Живой документ. Написан по итогам разработки лендинга AI-WORKSHOP.
Применять на любом следующем лендинге.

---

## Фаза 0 — Брифинг (до всего)

Одна страница, фиксирует:
- Продукт: что продаём, кому, почём
- Цель страницы: заявка / звонок / прямая оплата
- Дедлайн и кто принимает финальное решение
- Есть ли фото спикера / продукта, или нужны референсы
- Будет ли EN-версия

Без брифинга не начинаем. Брифинг = `plans/landing-brief.md`.

---

## Фаза 1 — Исследование

**Конкуренты и аналоги:**
- 5–7 сайтов прямых конкурентов (что у них на лендинге)
- 3–5 референсов красивых сайтов вне ниши — только для дизайна

**Скрины по блокам** (собрать папку):
- Hero / первый экран
- Секция с болями
- Секция «для кого»
- Программа / что внутри
- Спикер / эксперт
- Отзывы
- Цена / оффер
- FAQ
- Footer / CTA в конце

**Аудитория:**
- Кто покупает (роль, боль, уровень)
- Какие возражения — «дорого», «не моё», «посмотрю потом»
- Язык аудитории: какими словами они описывают проблему

Результат этой фазы: 1-2 страницы заметок + папка скринов.

---

## Фаза 2 — Контент (тексты)

**Структура страницы:**
- Список секций в порядке появления
- Название каждой секции
- Примерное количество блоков внутри

**Для каждой секции:**
- Заголовок (H2)
- Подзаголовок или вводная фраза
- Основной контент (буллеты / карточки / таймлайн)
- CTA (если есть)

**Правило:** все тексты одобрены экспертом / заказчиком **до верстки**.
Правки в тексте после верстки — потеря времени. Текст → одобрение → код.

Результат: `plans/landing-copy-ru.md` с финальными текстами.

---

## Фаза 3 — Дизайн-система

Фиксируем до написания CSS:

**Цвет:**
- Тёмный фон: `#0D0F14`
- Светлый фон: `#ECEDEF`
- Акцент: `#E8722A` (или цвет проекта)
- Текст первичный / вторичный (тёмная и светлая темы)

**Шрифты:**
- Display (заголовки): Unbounded / Montserrat / другой геометрический
- Body (текст): Wix Madefor Display / Inter / другой читаемый

**Компоненты:**
- Кнопка primary: 3D-тень + hover (светлеет) + active (нажимается вниз)
- Карточка тёмная: glass morphism + shimmer ::before при hover
- Карточка светлая: white + мягкая тень + shimmer при hover
- Иконки: rounded квадрат с оранжевым градиентом

**Эффекты:**
- 3D-наклон карточек (mousemove) — отключить на touch
- Shimmer-блик при hover
- Shimmer при тапе (mobile: :active)
- Счётчики с нуля при входе в вьюпорт (IntersectionObserver)
- Reveal-анимация секций при скролле

**Источник дизайна:**
- Только скрины от заказчика
- Не брать из других проектов (Agortex, другие сайты)
- Не брать из брендбука другого клиента

## Фаза 5 — HTML-сборка

### 5.1 Структура
```
landing/
├── index.html               ← основной файл
├── assets/
│   ├── css/
│   │   ├── tailwind.css     ← СКОМПИЛИРОВАННЫЙ, не CDN
│   │   └── custom.css       ← дизайн-система + компоненты
│   └── img/                 ← изображения
├── tailwind-input.css       ← @tailwind base/components/utilities
├── tailwind.config.js       ← конфиг с контентом и токенами
└── en/
    └── index.html           ← EN-версия (если нужна)
```

### 5.2 CSS-правила при сборке

**Tailwind — только компилированный:**
```bash
./node_modules/.bin/tailwindcss -c tailwind.config.js -i tailwind-input.css -o assets/css/tailwind.css --minify
```
Tailwind CDN (400KB JS) в продакшн — никогда.

**Google Fonts — через preconnect, не @import:**
```html
<link rel="preconnect" href="https://fonts.googleapis.com" />
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin />
<link rel="stylesheet" href="https://fonts.googleapis.com/css2?family=...&display=swap" />
```

**Каскад CSS — важно:**
custom.css загружается после tailwind.css → базовый класс `.metric-number { font-size: 3rem }` перебивает Tailwind-классы `text-xl`, `text-2xl` и т.д.
Если Tailwind-класс не работает — добавить compound selector с `!important` в конец custom.css:
```css
.metric-number.text-xl { font-size: 1.25rem !important; }
```

### 5.3 JavaScript
- Бургер-меню: DOMContentLoaded + touchstart + overlay + body scroll-lock + auto-close при скролле >60px
- Счётчики: IntersectionObserver, threshold 0.4, ease-out cubic, поддержка суффиксов `%` и `+`
- Reveal-анимация: threshold 0.05, исключить hero-секцию, fallback timeout 1200ms
- 3D-наклон карточек: проверять `matchMedia('(hover: none)')` — на touch не применять
- Форма: fetch POST, spinner, success/error state, блокировка двойного клика

## Фаза 7 — Мобильный аудит (375px, 390px, 414px)

**Это не доделка — это обязательная фаза.**

### Навигация
- [ ] Бургер открывается / закрывается
- [ ] Меню закрывается при тапе на ссылку
- [ ] Overlay затемняет страницу
- [ ] Скролл страницы заблокирован при открытом меню
- [ ] При скролле вниз на 60px меню закрывается

### Текст и типографика
- [ ] Заголовки читаемы (минимум 1.4rem на 375px)
- [ ] Body-текст минимум 15px
- [ ] Нет горизонтальной прокрутки (`overflow-x: hidden`)
- [ ] Длинные слова не выходят за границу

### Карточки и сетки
- [ ] Сетки перестраиваются в 1 колонку (не 2×2 на узком экране)
- [ ] Карточки не обрезаются по краям
- [ ] Минимальная высота кнопок 48px

### Touch-эффекты
- [ ] При тапе на карточку: shimmer + лёгкое нажатие (`:active`)
- [ ] 3D-наклон ОТКЛЮЧЁН (hover: none)
- [ ] Кнопки: `:active` нажимается вниз

### Медиа
- [ ] Видео-отзывы: горизонтальный swipe (карусель), не сетка
- [ ] Изображения не растягиваются

### Производительность на мобиле
- [ ] Нет CDN-скриптов > 50KB
- [ ] Страница открывается за < 3 сек на 3G (проверить Lighthouse)

### Реальное устройство
Обязательно открыть на iPhone или Android, не только DevTools.
DevTools не эмулирует touch-события и шрифтовые особенности Safari полностью.

## Фаза 9 — SEO и метаданные

- [ ] `<title>` — конкретный, с ключевым словом
- [ ] `<meta name="description">` — 150–160 символов
- [ ] `<meta property="og:title">`, `og:description`, `og:image`
- [ ] `lang="ru"` (или `en` для EN-версии)
- [ ] Favicon

## Стоп-правила

1. **Tailwind CDN** в продакшн — никогда (400KB JS, медленно на iPhone)
2. **Дизайн** — только от заказчика, не из других проектов
3. **Тексты** — одобрить до верстки
4. **Мобиль** — проверять на реальном устройстве, не только DevTools
5. **CSS каскад** — если Tailwind-класс не работает, проверить custom.css
6. **EN-версия** — отдельная сессия, не «по-быстрому» в конце
